Mastering the Favicon for Mobile Devices in 2026

If you have ever tapped 'Add to Home Screen' on your iPhone and groaned because your site saved as a blurry, unreadable miniature screenshot of your homepage—your favicon setup is incomplete. Mobile browsers do not care about your legacy desktop icons. They expect a specific set of high-resolution assets to make your website feel like a native app.

Look at GitHub's mobile web experience. When you view their site in Safari's tab grid, you see a crisp, perfectly padded icon. Save it to your Android device, and it perfectly adapts to whatever icon shape your OS uses. They do not achieve this by accident. They use a targeted strategy for their favicon for mobile devices.

Stop trying to serve fifteen different icon sizes. In 2026, you only need a few specific files to cover 99% of mobile devices. I am going to show you exactly how to set this up.

Prerequisites for Mobile Icons

Before writing any code, you need the right source image. Do not use a 32x32 pixel file. You need a high-resolution, square version of your logo. I recommend starting with a 512x512 pixel PNG or a clean SVG. If your logo has text, drop it. Use the standalone brand mark so it remains legible on small smartphone screens.

Step-by-Step Mobile Favicon Setup

Step 1: Conquer iOS with the Apple Touch Icon

Apple is notoriously stubborn. Safari on iOS completely ignores standard favicon declarations when saving to the home screen or displaying in the mobile Safari tab overview. It demands a specific file called the Apple Touch Icon.

You need exactly one 180x180 pixel PNG. Do not use an SVG here; iOS will ignore it. Furthermore, iOS fills any transparent background with black. If you have a dark logo on a transparent background, it will become an invisible black-on-black square. Always use a solid background color for this specific file.

You can read more about the nuances of this in our Apple Touch Icon guide, but the implementation is just one line of HTML.

Step 2: Tame Android with the Web App Manifest

Chrome on Android handles icons entirely differently. Instead of looking for specific HTML tags, it looks for a JSON file called a web app manifest. This file tells Android how your site should behave when installed or viewed in the mobile task switcher.

For Android, you need two PNG sizes: 192x192 and 512x512. You define these inside a site.webmanifest file that lives in your root directory.

Step 3: Implement Maskable Icons

Android OEMs love different icon shapes. Samsung might use squircles, Pixel phones might use circles, and others use teardrops. If you do not provide a maskable icon, Android takes your square logo and shrinks it inside a white circle, which looks terrible.

By adding the purpose: 'maskable' property to your manifest, you give Android permission to crop your icon to fit the system's native shape. Just ensure your logo's main graphic is safely within the center 'safe zone' (the inner 80% of the image).

Step 4: The Final Code Integration

Here is the exact code you need to tie your favicon for mobile devices together. First, add this to your HTML <head> section:

<!-- iOS Safari -->
<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

<!-- Android / Chrome -->
<link rel='manifest' href='/site.webmanifest'>

<!-- Modern Mobile Browsers (SVG Fallback) -->
<link rel='icon' type='image/svg+xml' href='/icon.svg'>

Next, create your site.webmanifest file with the following JSON structure:

{
  'name': 'Your App Name',
  'short_name': 'App',
  'icons': [
    {
      'src': '/icon-192x192.png',
      'sizes': '192x192',
      'type': 'image/png'
    },
    {
      'src': '/icon-512x512.png',
      'sizes': '512x512',
      'type': 'image/png',
      'purpose': 'any maskable'
    }
  ],
  'theme_color': '#ffffff',
  'background_color': '#ffffff',
  'display': 'standalone'
}

Common Pitfalls to Avoid

  • Ignoring the iOS transparency rule: As mentioned, transparent PNGs turn into black squares on iPhones. Always bake a solid background color into your apple-touch-icon.png.
  • Bloating the head tag: You do not need 57x57, 72x72, or 114x114 icons anymore. Modern iOS devices scale the 180x180 image perfectly. Delete the legacy bloat.
  • Forgetting the manifest theme color: The theme_color in your manifest dictates the color of the Android status bar when your site is launched from the home screen. Do not leave it blank.

ステップ1: Apple Touch IconでiOSを攻略する

この詳細については、Apple Touch Iconガイドで解説していますが、実装自体はHTML1行で済みます。

AndroidのChromeは、アイコンの扱いがまったく異なります。特定のHTMLタグを探すのではなく、「Web App Manifest」と呼ばれるJSONファイルを探します。このファイルは、サイトがインストールされたときやタスクスイッチャーで表示されたときの挙動をAndroidに伝えます。

Android向けには、192x192と512x512の2つのPNGサイズが必要です。これらをルートディレクトリにあるsite.webmanifestファイル内で定義します。

Androidのメーカーは、さまざまなアイコンの形状を好みます。Samsungは角丸四角形、Pixelは円形、他のブランドはしずく型かもしれません。Maskable Iconを提供しないと、Androidはあなたの正方形のロゴを縮小し、白い円の中に押し込んでしまいます。これは非常に見栄えが悪いです。

マニフェストにpurpose: 'maskable'プロパティを追加することで、システムのネイティブな形状に合わせてアイコンをトリミングする許可をAndroidに与えることができます。ロゴのメイングラフィックが、中央の「セーフゾーン」（画像の内側80%）に収まっていることだけ確認してください。

단계별 모바일 파비콘 설정

1단계: Apple Touch Icon으로 iOS 정복하기

Apple은 고집이 세기로 유명합니다. iOS의 Safari는 홈 화면에 저장하거나 모바일 Safari 탭 개요에 표시할 때 표준 파비콘 선언을 완전히 무시합니다. 대신 Apple Touch Icon이라는 특정 파일을 요구합니다.

정확히 180x180 픽셀 PNG 하나가 필요합니다. 여기서 SVG를 사용하지 마세요. iOS는 이를 무시합니다. 게다가 iOS는 투명한 배경을 모두 검은색으로 채웁니다. 투명한 배경에 어두운 로고가 있다면 보이지 않는 검은색 사각형이 될 것입니다. 이 특정 파일에는 항상 단색 배경을 사용하세요.

이에 대한 뉘앙스는 Apple Touch Icon 가이드에서 더 자세히 읽을 수 있지만, 구현은 HTML 한 줄이면 됩니다.

2단계: Web App Manifest로 Android 길들이기

Android의 Chrome은 아이콘을 완전히 다르게 처리합니다. 특정 HTML 태그를 찾는 대신 웹 앱 매니페스트라는 JSON 파일을 찾습니다. 이 파일은 사이트가 설치되거나 모바일 작업 전환기에서 표시될 때 어떻게 동작해야 하는지 Android에 알려줍니다.

Android의 경우 192x192와 512x512 두 가지 PNG 크기가 필요합니다. 루트 디렉토리에 있는 site.webmanifest 파일 안에 이를 정의합니다.

3단계: Maskable Icon(마스크 가능한 아이콘) 구현하기

Android 제조사들은 다양한 아이콘 모양을 좋아합니다. 삼성은 둥근 사각형(squircle)을, Pixel 폰은 원형을, 다른 기기들은 물방울 모양을 사용할 수 있습니다. 마스크 가능한 아이콘을 제공하지 않으면 Android는 정사각형 로고를 축소하여 흰색 원 안에 넣는데, 이는 매우 보기 흉합니다.

매니페스트에 purpose: 'maskable' 속성을 추가하면 Android가 시스템의 기본 모양에 맞게 아이콘을 자를 수 있도록 허용하는 것입니다. 로고의 메인 그래픽이 중앙의 '안전 영역'(이미지 내부 80%) 안에 안전하게 위치하는지 확인하기만 하면 됩니다.

Configuración paso a paso del Favicon Móvil

Paso 1: Conquista iOS con el Apple Touch Icon

Apple es notoriamente terco. Safari en iOS ignora por completo las declaraciones estándar de favicon cuando se guarda en la pantalla de inicio o se muestra en la vista general de pestañas de Safari móvil. Exige un archivo específico llamado Apple Touch Icon.

Necesitas exactamente un PNG de 180x180 píxeles. No uses un SVG aquí; iOS lo ignorará. Además, iOS rellena cualquier fondo transparente con negro. Si tienes un logotipo oscuro sobre un fondo transparente, se convertirá en un cuadrado negro sobre negro invisible. Usa siempre un color de fondo sólido para este archivo específico.

Puedes leer más sobre los matices de esto en nuestra guía de Apple Touch Icon, pero la implementación es solo una línea de HTML.

Paso 2: Doma Android con el Web App Manifest

Chrome en Android maneja los iconos de manera completamente diferente. En lugar de buscar etiquetas HTML específicas, busca un archivo JSON llamado web app manifest. Este archivo le dice a Android cómo debe comportarse tu sitio cuando se instala o se ve en el selector de tareas móvil.

Para Android, necesitas dos tamaños de PNG: 192x192 y 512x512. Defines estos dentro de un archivo site.webmanifest que vive en tu directorio raíz.

Paso 3: Implementa Iconos Enmascarables (Maskable Icons)

A los fabricantes de Android les encantan las diferentes formas de iconos. Samsung podría usar 'squircles' (cuadrados redondeados), los teléfonos Pixel podrían usar círculos, y otros usan formas de lágrima. Si no proporcionas un icono enmascarable, Android toma tu logotipo cuadrado y lo encoge dentro de un círculo blanco, lo cual se ve terrible.

Al agregar la propiedad purpose: 'maskable' a tu manifiesto, le das permiso a Android para recortar tu icono y ajustarlo a la forma nativa del sistema. Solo asegúrate de que el gráfico principal de tu logotipo esté de forma segura dentro de la 'zona segura' central (el 80% interior de la imagen).

Paso 4: La Integración Final del Código

Aquí tienes el código exacto que necesitas para unir tu favicon para dispositivos móviles. Primero, agrega esto a tu sección <head> de HTML:

<!-- iOS Safari -->
<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

<!-- Android / Chrome -->
<link rel='manifest' href='/site.webmanifest'>

<!-- Navegadores Móviles Modernos (Respaldo SVG) -->
<link rel='icon' type='image/svg+xml' href='/icon.svg'>

A continuación, crea tu archivo site.webmanifest con la siguiente estructura JSON:

{
  'name': 'Nombre de tu App',
  'short_name': 'App',
  'icons': [
    {
      'src': '/icon-192x192.png',

      'type': 'image/png'
    },
    {
      'src': '/icon-512x512.png',

      'type': 'image/png',
      'purpose': 'any maskable'
    }
  ],
  'theme_color': '#ffffff',
  'background_color': '#ffffff',

Errores Comunes a Evitar

  • Ignorar la regla de transparencia de iOS: Como se mencionó, los PNG transparentes se convierten en cuadrados negros en los iPhones. Siempre integra un color de fondo sólido en tu apple-touch-icon.png.
  • Inflar la etiqueta head: Ya no necesitas iconos de 57x57, 72x72 o 114x114. Los dispositivos iOS modernos escalan la imagen de 180x180 perfectamente. Elimina ese código heredado.
  • Olvidar el color del tema del manifiesto: El theme_color en tu manifiesto dicta el color de la barra de estado de Android cuando tu sitio se lanza desde la pantalla de inicio. No lo dejes en blanco.
