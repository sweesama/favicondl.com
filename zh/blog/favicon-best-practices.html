2026 年 Favicon 最佳实践

Getting favicons right in 2026 means balancing modern features with backward compatibility. This guide distills everything into an actionable checklist.

The 2026 Favicon Checklist

  1. favicon.ico (32×32) — Place at root. Universal fallback.
  2. favicon-32x32.png — For modern browser tabs (Retina).
  3. favicon-16x16.png — For standard-DPI browser tabs.
  4. apple-touch-icon.png (180×180) — For iOS home screen.
  5. android-chrome-192x192.png — For Android and PWA.
  6. android-chrome-512x512.png — For PWA splash screens.
  7. site.webmanifest — PWA manifest referencing the Android icons.
  8. HTML <link> tags — In every page's <head>.

Design Best Practices

Keep It Simple

At 16×16 pixels, fine details disappear entirely. The best favicons are:

  • A single letter or symbol
  • High contrast between foreground and background
  • Bold, clean shapes without gradients or thin lines

Use Transparent Backgrounds

Solid colored backgrounds look outdated in modern browser UIs. Transparent PNGs blend naturally with any browser theme — light mode, dark mode, or custom themes.

Exception: Apple Touch Icons should have a solid background because iOS fills transparent areas with black.

Design for Both Light and Dark

With dark mode becoming standard, test your favicon on both light and dark backgrounds. A dark favicon on a dark browser tab is invisible. Consider using an SVG favicon with prefers-color-scheme support (see our SVG favicon guide).

Start Large, Scale Down

Always design at 512×512 or larger, then export at each required size. Hand-optimize the 16×16 version if needed — automatic downscaling sometimes produces muddy results.

Technical Best Practices

File Placement

  • Place favicon.ico at the root of your domain — browsers look there automatically
  • Other files can go in a /favicons/ subdirectory if you prefer organization
  • Use absolute paths in your HTML (/favicon.ico, not favicon.ico)

Caching Strategy

  • Set long cache headers (1 year) for favicon files — they rarely change
  • When you do update, use a cache-busting query string: href="/favicon.ico?v=2"
  • Or rename the file: href="/favicon-v2.ico"

Performance

  • Optimize PNG files with tools like TinyPNG or pngquant
  • A 32×32 PNG should be under 2KB
  • Don't serve a 512×512 image when the browser only needs 32×32

SEO Best Practices

Google Search Results

Google shows favicons in search results. To ensure yours appears:

  • Provide a favicon that is a multiple of 48px (48×48, 96×96, etc.)
  • Don't block favicon files in robots.txt
  • Favicon must be on the same domain as the website
  • Must be a square image
  • Must not be inappropriate, offensive, or a generic icon

Consistency Across Pages

Use the same favicon on all pages of your site. Different favicons per page confuses users and dilutes brand recognition.

Common Anti-Patterns to Avoid

Anti-PatternWhy It's BadWhat to Do Instead
Only providing favicon.icoMissing iOS/Android/PWA supportProvide full set of sizes
Using a full logo with textUnreadable at 16pxUse a symbol or monogram
Renaming JPEG to .icoBrowser may reject itUse proper format conversion
Serving favicon from CDN onlyGoogle may not index itHost on same domain
Animated faviconsDistracting, poor supportUse a static icon
No favicon at allUnprofessional, hurts SEOAdd one — it takes 5 minutes

Testing Your Favicon Setup

  1. Browser tab — Check Chrome, Firefox, Safari, Edge
  2. Bookmarks — Bookmark the page and check the icon
  3. iOS home screen — Add to home screen on iPhone
  4. Android home screen — Add to home screen on Android
  5. Google Search — Use Search Console to check favicon indexing
  6. Social sharing — Check OG image for link previews

设计最佳实践

保持简洁

在 16×16 像素下，精细细节完全消失。最好的 Favicon 是：

  • 单个字母或符号
  • 前景和背景之间高对比度
  • 粗体、干净的形状，没有渐变或细线

使用透明背景

纯色背景在现代浏览器 UI 中显得过时。透明 PNG 能自然融入任何浏览器主题 — 浅色模式、暗黑模式或自定义主题。

例外：Apple Touch Icon 应该使用纯色背景，因为 iOS 会将透明区域填充为黑色。

为浅色和暗黑模式都设计

暗黑模式已经成为标配，测试你的 Favicon 在浅色和暗色背景上的效果。暗色 Favicon 在暗色浏览器标签页上是看不见的。考虑使用支持 prefers-color-scheme 的 SVG Favicon（参见我们的SVG Favicon 指南）。

从大图开始，逐步缩小

始终在 512×512 或更大尺寸设计，然后导出为各个所需尺寸。如有需要手动优化 16×16 版本 — 自动缩小有时会产生模糊的结果。

技术最佳实践

文件位置

  • 将 favicon.ico 放在域名根目录 — 浏览器会自动在那里查找
  • 其他文件可以放在 /favicons/ 子目录中以保持整洁
  • 在 HTML 中使用绝对路径（/favicon.ico，而非 favicon.ico）

缓存策略

  • 为 Favicon 文件设置长缓存头（1 年）— 它们很少变化
  • 更新时使用缓存破坏查询字符串：href="/favicon.ico?v=2"
  • 或重命名文件：href="/favicon-v2.ico"

2026年にfaviconを適切に設定するには、最新の機能と後方互換性のバランスを取る必要があります。このガイドでは、すべてを実用的なチェックリストにまとめました。

2026年版 Faviconチェックリスト

  1. favicon.ico (32×32) — ルートに配置します。普遍的なフォールバックです。
  2. favicon-32x32.png — 最新のブラウザタブ用（Retina対応）。
  3. favicon-16x16.png — 標準DPIのブラウザタブ用。
  4. apple-touch-icon.png (180×180) — iOSホーム画面用。
  5. android-chrome-192x192.png — AndroidおよびPWA用。
  6. android-chrome-512x512.png — PWAスプラッシュスクリーン用。
  7. site.webmanifest — Androidアイコンを参照するPWA manifest。
  8. HTML <link> tags — すべてのページの<head>内に記述します。

デザインのベストプラクティス

シンプルに保つ

16×16ピクセルでは、細かいディテールは完全に消えてしまいます。最適なfaviconは次のとおりです。

  • 一文字または記号
  • 前景と背景のコントラストが高い
  • グラデーションや細い線がない、太くクリーンな形状

透明な背景を使用する

単色の背景は、最新のブラウザUIでは古く見えます。透明なPNGは、ライトモード、ダークモード、カスタムテーマなど、あらゆるブラウザテーマに自然に溶け込みます。

例外: Apple Touch Iconsは、iOSが透明な領域を黒で埋めるため、単色の背景を持つ必要があります。

ライトモードとダークモードの両方に対応してデザインする

ダークモードが標準になりつつあるため、ライトとダークの両方の背景でfaviconをテストしてください。暗いfaviconが暗いブラウザタブにあると見えません。prefers-color-schemeをサポートするSVG faviconの使用を検討してください（当社のSVG faviconガイドを参照してください）。

大きなサイズから始めて、縮小する

常に512×512ピクセル以上でデザインし、必要な各サイズでエクスポートしてください。必要に応じて16×16バージョンを手動で最適化してください。自動縮小では、ぼやけた結果になることがあります。

技術的なベストプラクティス

ファイルの配置

  • favicon.icoはドメインのルートに配置してください。ブラウザは自動的にそこを探します。
  • 他のファイルは、整理のために/favicons/サブディレクトリに配置しても構いません。
  • HTMLでは絶対パスを使用してください（/favicon.icoであり、favicon.icoではありません）。

キャッシュ戦略

  • faviconファイルには長いキャッシュヘッダー（1年）を設定してください。これらはめったに変更されません。
  • 更新する際は、キャッシュを無効にするクエリ文字列を使用してください: href="/favicon.ico?v=2"
  • またはファイルをリネームしてください: href="/favicon-v2.ico"

パフォーマンス

  • TinyPNGやpngquantのようなツールでPNGファイルを最適化してください。
  • 32×32のPNGは2KB未満であるべきです。
  • ブラウザが32×32しか必要としないときに、512×512の画像を提供しないでください。

SEOのベストプラクティス

Google検索結果

Googleは検索結果にfaviconを表示します。あなたのfaviconが表示されるようにするには：

  • 48pxの倍数（48×48、96×96など）のfaviconを提供してください。
  • robots.txtでfaviconファイルをブロックしないでください。
  • faviconはウェブサイトと同じドメインにある必要があります。
  • 正方形の画像である必要があります。
  • 不適切、攻撃的、または一般的なアイコンであってはなりません。

ページ間の一貫性

サイトのすべてのページで同じfaviconを使用してください。ページごとに異なるfaviconを使用すると、ユーザーを混乱させ、ブランド認知度を低下させます。

디자인 모범 사례

단순하게 유지하기

16×16 픽셀에서는 미세한 디테일이 완전히 사라집니다. 최고의 favicon은 다음과 같습니다.

  • 단일 문자 또는 기호
  • 전경과 배경 간의 높은 대비
  • 그라데이션이나 얇은 선이 없는 굵고 깔끔한 모양

투명 배경 사용하기

단색 배경은 최신 브라우저 UI에서 구식으로 보입니다. 투명 PNG는 라이트 모드, 다크 모드 또는 사용자 지정 테마 등 모든 브라우저 테마와 자연스럽게 어울립니다.

예외: Apple Touch Icon은 iOS가 투명한 영역을 검은색으로 채우기 때문에 단색 배경을 가져야 합니다.

라이트 모드와 다크 모드 모두를 위해 디자인하기

다크 모드가 표준이 됨에 따라, 밝은 배경과 어두운 배경 모두에서 favicon을 테스트하십시오. 어두운 브라우저 탭에 어두운 favicon이 있으면 보이지 않습니다.prefers-color-scheme 지원이 있는 SVG favicon 사용을 고려해 보십시오 (저희 SVG favicon 가이드를 참조하십시오).

크게 시작하여 축소하기

항상 512×512 이상으로 디자인한 다음, 필요한 각 크기로 내보내십시오. 필요한 경우 16×16 버전을 수동으로 최적화하십시오. 자동 축소는 때때로 흐릿한 결과를 초래할 수 있습니다.

기술 모범 사례

파일 배치

  • favicon.ico는 도메인의 루트에 배치하십시오. 브라우저가 자동으로 그곳을 찾습니다.
  • 다른 파일은 정리를 위해 /favicons/ 하위 디렉토리에 넣을 수 있습니다.
  • HTML에서 절대 경로를 사용하십시오 (/favicon.ico, favicon.ico가 아님).

캐싱 전략

  • favicon 파일에 긴 캐시 헤더(1년)를 설정하십시오. 이 파일들은 거의 변경되지 않습니다.
  • 업데이트할 때는 캐시 무효화 쿼리 문자열을 사용하십시오: href="/favicon.ico?v=2"
  • 또는 파일 이름을 변경하십시오: href="/favicon-v2.ico"

성능

  • TinyPNG 또는 pngquant와 같은 도구로 PNG 파일을 최적화하십시오.
  • 32×32 PNG는 2KB 미만이어야 합니다.
  • 브라우저가 32×32만 필요할 때 512×512 이미지를 제공하지 마십시오.

SEO 모범 사례

Google 검색 결과

Google은 검색 결과에 favicon을 표시합니다. 당신의 favicon이 나타나도록 하려면:

  • 48px의 배수(48×48, 96×96 등)인 favicon을 제공하십시오.
  • robots.txt에서 favicon 파일을 차단하지 마십시오.
  • favicon은 웹사이트와 동일한 도메인에 있어야 합니다.
  • 정사각형 이미지여야 합니다.
  • 부적절하거나, 불쾌하거나, 일반적인 아이콘이 아니어야 합니다.

페이지 간 일관성

사이트의 모든 페이지에서 동일한 favicon을 사용하십시오. 페이지마다 다른 favicon을 사용하면 사용자를 혼란스럽게 하고 브랜드 인지도를 희석시킵니다.

Configurar los favicons correctamente en 2026 significa equilibrar las características modernas con la compatibilidad con versiones anteriores. Esta guía resume todo en una lista de verificación práctica.

La lista de verificación de Favicon para 2026

  1. favicon.ico (32×32) — Colócalo en la raíz. Es el respaldo universal.
  2. favicon-32x32.png — Para pestañas de navegadores modernos (Retina).
  3. favicon-16x16.png — Para pestañas de navegadores con DPI estándar.
  4. apple-touch-icon.png (180×180) — Para la pantalla de inicio de iOS.
  5. android-chrome-192x192.png — Para Android y PWA.
  6. android-chrome-512x512.png — Para pantallas de bienvenida de PWA.
  7. site.webmanifest — El manifest de PWA que referencia los iconos de Android.
  8. HTML <link> tags — En el <head> de cada página.

Mejores prácticas de diseño

Mantenlo simple

Con 16×16 píxeles, los detalles finos desaparecen por completo. Los mejores favicons son:

  • Una sola letra o símbolo
  • Alto contraste entre el primer plano y el fondo
  • Formas audaces y limpias sin degradados ni líneas finas

Usa fondos transparentes

Los fondos de color sólido se ven anticuados en las interfaces de usuario de los navegadores modernos. Los PNG transparentes se integran naturalmente con cualquier tema del navegador, ya sea modo claro, modo oscuro o temas personalizados.

Excepción: Los Apple Touch Icons deben tener un fondo sólido porque iOS rellena las áreas transparentes con negro.

Diseña para claro y oscuro

Con el modo oscuro convirtiéndose en estándar, prueba tu favicon tanto en fondos claros como oscuros. Un favicon oscuro en una pestaña de navegador oscura es invisible. Considera usar un favicon SVG con soporte para prefers-color-scheme (consulta nuestra guía de favicons SVG).

Empieza grande, escala hacia abajo

Diseña siempre a 512×512 o más grande, luego exporta a cada tamaño requerido. Optimiza manualmente la versión de 16×16 si es necesario; la reducción automática a veces produce resultados borrosos.

Mejores prácticas técnicas

Ubicación de archivos

  • Coloca favicon.ico en la raíz de tu dominio; los navegadores lo buscan automáticamente.
  • Otros archivos pueden ir en un subdirectorio /favicons/ si prefieres organización.
  • Usa rutas absolutas en tu HTML (/favicon.ico, no favicon.ico).

Estrategia de caché

  • Establece encabezados de caché largos (1 año) para los archivos de favicon; rara vez cambian.
  • Cuando actualices, usa una cadena de consulta para invalidar la caché: href="/favicon.ico?v=2"
  • O renombra el archivo: href="/favicon-v2.ico"

Rendimiento

  • Optimiza los archivos PNG con herramientas como TinyPNG o pngquant.
  • Un PNG de 32×32 debería pesar menos de 2KB.
  • No sirvas una imagen de 512×512 cuando el navegador solo necesita 32×32.

Mejores prácticas de SEO

Resultados de búsqueda de Google

Google muestra favicons en los resultados de búsqueda. Para asegurarte de que el tuyo aparezca:

  • Proporciona un favicon que sea un múltiplo de 48px (48×48, 96×96, etc.).
  • No bloquees los archivos de favicon en robots.txt.
  • El favicon debe estar en el mismo dominio que el sitio web.
  • Debe ser una imagen cuadrada.
  • No debe ser inapropiado, ofensivo o un icono genérico.

Consistencia entre páginas

Usa el mismo favicon en todas las páginas de tu sitio. Favicons diferentes por página confunden a los usuarios y diluyen el reconocimiento de la marca.
