如何正确设置添加到主屏幕的 Favicon 图标 (2026指南)

Ever tapped 'Share' then 'Add to Home Screen' on your iPhone, expecting a slick app icon, but got a miniature, illegible screenshot of the webpage instead? I see this on production sites constantly. Your desktop tabs might look fine, but your mobile add to home screen icon favicon setup is likely incomplete.

Look at Stripe or GitHub. When you save their web apps to your phone, the icon sits on your home screen looking exactly like a native app downloaded from the App Store. They don't leave this to chance or rely on browser fallbacks. They explicitly define what the operating system should display.

Mobile browsers are notoriously lazy. If you don't explicitly tell iOS Safari or Android Chrome which image to use for the home screen shortcut, they panic. Safari takes a literal screenshot of your site. Chrome tries to stretch your tiny 32x32 desktop favicon into a massive 192x192 square. Both results look terrible and destroy your brand's credibility.

The 3-Step Setup for Home Screen Perfection

You don't need a massive block of legacy HTML tags to fix this. In 2026, we only need to satisfy two major players: Apple (iOS) and Google (Android). Here is exactly how to do it.

Step 1: Create the Apple Touch Icon (iOS)

Apple stubbornly ignores standard web manifests for home screen icons. You have to feed Safari a specific link tag. The image must be a 180x180 pixel PNG.

(Do not use transparency here — iOS fills transparent areas with solid black, making your logo look like a ransom note. Always use a solid background color).

<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

Place this tag right inside your <head>. For a deeper dive into iOS quirks, check our Apple Touch Icon setup guide.

Step 2: Configure the Web Manifest (Android)

Android devices look for a site.webmanifest file. This JSON file tells Chrome exactly how your site should behave when saved to the home screen, including which icon to use and whether to hide the browser UI.

Create a file named site.webmanifest in your root directory and add this code:

{
  "name": "Your App Name",
  "short_name": "App",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone"
}

Notice the display: standalone property. This is the magic line that removes the URL bar when a user opens your site from their home screen, giving it that native app feel.

Step 3: Link the Manifest in Your HTML

You need to point your HTML document to that manifest file. Add this right below your Apple touch icon tag:

<link rel='manifest' href='/site.webmanifest'>

Common Pitfalls to Avoid

Even senior developers mess this up. Here are the traps I see people fall into every week:

  • Using SVGs for iOS: Safari still doesn't fully support SVG for the home screen icon. Stick to PNG. Read our SVG favicon guide to see where SVGs actually belong (hint: desktop tabs).
  • Forgetting the background color: As mentioned, iOS hates transparency. Fill your PNG background with a solid brand color.
  • Caching nightmares: If you update your icon, iOS will stubbornly hold onto the old one. Append a version query string like ?v=2 to the file path in your HTML to force an update.

Testing Your Implementation

Don't trust your desktop browser's dev tools for this. The only real way to test your add to home screen icon favicon is on a physical device.

Open your site on an iPhone, tap the Share button, and select 'Add to Home Screen'. Look at the preview icon. If it's your crisp 180x180 PNG, you win. Do the same on an Android device using Chrome's menu. If the icon looks pixelated, double-check your manifest paths.

Don't overcomplicate this process. You need exactly three files in 2026: a standard desktop icon, a 180px Apple Touch Icon, and a manifest with Android icons. 完美主屏幕图标的 3 步设置法

你不需要写一大堆过时的 HTML 标签来解决这个问题。在 2026 年，我们只需要搞定两大阵营：Apple (iOS) 和 Google (Android)。以下是具体做法。

第 1 步：创建 Apple Touch Icon (iOS)

Apple 固执地无视了标准的 Web Manifest 主屏幕图标规范。你必须专门给 Safari 喂一个特定的 link 标签。这张图片必须是 180x180 像素的 PNG。

（千万别在这里用透明背景——iOS 会把透明区域自动填充为纯黑色，让你的 Logo 看起来像个黑洞。一定要使用纯色背景）。

<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

把这行代码放在你的 <head> 里。想深入了解 iOS 的各种奇葩设定，可以看看我们的 Apple Touch Icon 设置指南。

第 2 步：配置 Web Manifest (Android)

Android 设备会寻找 site.webmanifest 文件。这个 JSON 文件告诉 Chrome，当你的网站被保存到主屏幕时应该如何表现，包括使用哪个图标，以及是否隐藏浏览器 UI。

在你的根目录下创建一个名为 site.webmanifest 的文件，并添加以下代码：

{
  "name": "你的应用名称",
  "short_name": "应用",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone"
}

注意 display: standalone 这个属性。这就是那个能让用户从主屏幕打开你的网站时隐藏 URL 地址栏的魔法代码，让它拥有原生 App 的体验。

第 3 步：在 HTML 中链接 Manifest

你需要让 HTML 文档指向那个 manifest 文件。把它加在 Apple touch icon 标签的下面：

<link rel='manifest' href='/site.webmanifest'>

需要避免的常见坑

即使是资深开发者也经常在这里翻车。以下是我每周都会看到的错误：

  • 在 iOS 上使用 SVG： Safari 至今仍未完全支持将 SVG 用作主屏幕图标。老老实实用 PNG 吧。阅读我们的 SVG favicon 指南 看看 SVG 真正该用在哪（提示：桌面端标签页）。
  • 忘记设置背景色： 如前所述，iOS 讨厌透明度。用你的品牌色填满 PNG 的背景。
  • 缓存噩梦： 如果你更新了图标，iOS 会死死抱住旧图标不放。在 HTML 的文件路径后面加上版本查询字符串（比如 ?v=2）来强制更新。

别太相信桌面浏览器的开发者工具。测试“添加到主屏幕”图标的唯一真实方法是在真机上操作。

别把这个过程搞得太复杂。在 2026 年，你只需要三个文件：一个标准的桌面图标，一个 180px 的 Apple Touch Icon，以及一个包含 Android 图标的 manifest。如果你需要从一张高清 Logo 中一次性生成这些确切的文件，把它扔进 Mzu favicondl，然后把输出的文件丢进你的根目录即可，它会自动处理好所有尺寸和格式。

完璧なホーム画面アイコンのための3ステップ

これを修正するために、レガシーなHTMLタグを大量に記述する必要はありません。2026年現在、Apple（iOS）とGoogle（Android）の2大巨頭を満足させるだけで十分です。具体的な手順は以下の通りです。

ステップ1: Apple Touch Iconの作成（iOS）

Appleはホーム画面アイコンに関する標準のWeb Manifestを頑なに無視します。Safariには専用のlinkタグを与える必要があります。画像は180x180ピクセルのPNGでなければなりません。

（ここで透過背景を使用しないでください。iOSは透過部分を真っ黒に塗りつぶすため、ロゴが不気味に見えてしまいます。必ずベタ塗りの背景色を使用してください）。

<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

このタグを<head>内に配置します。iOS特有の癖についてさらに詳しく知りたい場合は、Apple Touch Icon設定ガイドをご覧ください。

ステップ2: Web Manifestの設定（Android）

Androidデバイスはsite.webmanifestファイルを探します。このJSONファイルは、サイトがホーム画面に保存されたときの動作（どのアイコンを使用するか、ブラウザのUIを隠すかどうかなど）をChromeに正確に伝えます。

ルートディレクトリにsite.webmanifestという名前のファイルを作成し、以下のコードを追加します：

{
  "name": "あなたのアプリ名",
  "short_name": "アプリ",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone"
}

display: standaloneプロパティに注目してください。これは、ユーザーがホーム画面からサイトを開いたときにURLバーを消し、ネイティブアプリのような感覚を与える魔法のコードです。

ステップ3: HTMLでManifestをリンクする

HTMLドキュメントからそのmanifestファイルを参照する必要があります。Apple touch iconタグのすぐ下にこれを追加します：

<link rel='manifest' href='/site.webmanifest'>

避けるべきよくある落とし穴

シニアエンジニアでさえ、ここでミスをすることがあります。私が毎週のように目にする罠をいくつか紹介します：

  • iOSでSVGを使用する： Safariはホーム画面アイコンとしてのSVGをまだ完全にはサポートしていません。PNGを使用してください。SVGが本当に適している場所については、SVG faviconガイドをお読みください。
  • 背景色を忘れる： 前述の通り、iOSは透過を嫌います。PNGの背景はブランドカラーで塗りつぶしてください。
  • キャッシュの悪夢： アイコンを更新しても、iOSは古いアイコンを頑なに保持しようとします。強制的に更新させるには、HTMLのファイルパスに?v=2のようなバージョンクエリストリングを追加します。

実装のテスト

デスクトップブラウザの開発者ツールを信用しないでください。ホーム画面追加アイコンをテストする唯一の確実な方法は、実機を使用することです。

iPhoneでサイトを開き、共有ボタンをタップして「ホーム画面に追加」を選択します。プレビューアイコンを確認してください。それが鮮明な180x180のPNGであれば成功です。AndroidデバイスでもChromeのメニューを使って同じことを行います。アイコンがピクセル化して見える場合は、manifestのパスを再確認してください。

완벽한 홈 화면 아이콘을 위한 3단계 설정

이를 해결하기 위해 오래된 HTML 태그를 잔뜩 작성할 필요는 없습니다. 2026년 현재, 우리는 Apple(iOS)과 Google(Android)이라는 두 주요 플랫폼만 만족시키면 됩니다. 정확한 방법은 다음과 같습니다.

1단계: Apple Touch Icon 생성 (iOS)

Apple은 홈 화면 아이콘에 대한 표준 웹 매니페스트(Web Manifest)를 고집스럽게 무시합니다. Safari에는 반드시 특정 link 태그를 제공해야 합니다. 이미지는 180x180 픽셀의 PNG여야 합니다.

(여기서 투명 배경을 사용하지 마세요. iOS는 투명한 영역을 검은색으로 채워버리기 때문에 로고가 흉측하게 보일 수 있습니다. 항상 단색 배경을 사용하세요).

<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

이 태그를 <head> 안에 넣으세요. iOS의 독특한 특성에 대해 더 깊이 알고 싶다면 Apple Touch Icon 설정 가이드를 확인하세요.

2단계: 웹 매니페스트 구성 (Android)

Android 기기는 site.webmanifest 파일을 찾습니다. 이 JSON 파일은 사이트가 홈 화면에 저장될 때 어떤 아이콘을 사용할지, 브라우저 UI를 숨길지 등 Chrome이 어떻게 동작해야 하는지 정확히 알려줍니다.

루트 디렉토리에 site.webmanifest라는 파일을 만들고 다음 코드를 추가하세요:

{
  "name": "여러분의 앱 이름",
  "short_name": "앱",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone"
}

display: standalone 속성에 주목하세요. 사용자가 홈 화면에서 사이트를 열었을 때 URL 표시줄을 없애고 네이티브 앱 같은 느낌을 주는 마법의 코드입니다.

3단계: HTML에 매니페스트 연결

HTML 문서가 해당 매니페스트 파일을 가리키도록 해야 합니다. Apple touch icon 태그 바로 아래에 다음을 추가하세요:

<link rel='manifest' href='/site.webmanifest'>

피해야 할 흔한 실수들

시니어 개발자들도 여기서 실수를 하곤 합니다. 제가 매주 목격하는 흔한 함정들은 다음과 같습니다:

  • iOS에 SVG 사용하기: Safari는 아직 홈 화면 아이콘으로 SVG를 완벽하게 지원하지 않습니다. PNG를 고수하세요. SVG가 실제로 어디에 쓰여야 하는지는 SVG 파비콘 가이드를 읽어보세요.
  • 배경색 잊기: 앞서 언급했듯, iOS는 투명도를 싫어합니다. PNG 배경을 브랜드 컬러로 채우세요.
  • 캐싱의 악몽: 아이콘을 업데이트하더라도 iOS는 이전 아이콘을 고집스럽게 유지하려 합니다. HTML의 파일 경로에 ?v=2와 같은 버전 쿼리 문자열을 추가하여 강제로 업데이트하세요.

구현 테스트하기

데스크톱 브라우저의 개발자 도구를 너무 믿지 마세요. 홈 화면 추가 아이콘을 테스트하는 유일한 진짜 방법은 실제 기기를 사용하는 것입니다.

La configuración de 3 pasos para la perfección en la pantalla de inicio

No necesitas un bloque masivo de etiquetas HTML heredadas para arreglar esto. En 2026, solo necesitamos satisfacer a dos grandes jugadores: Apple (iOS) y Google (Android). Aquí te explico exactamente cómo hacerlo.

Paso 1: Crea el Apple Touch Icon (iOS)

Apple ignora obstinadamente los manifiestos web estándar para los íconos de la pantalla de inicio. Tienes que darle a Safari una etiqueta link específica. La imagen debe ser un PNG de 180x180 píxeles.

(No uses transparencia aquí — iOS rellena las áreas transparentes con negro sólido, haciendo que tu logo parezca una nota de rescate. Usa siempre un color de fondo sólido).

<link rel='apple-touch-icon' sizes='180x180' href='/apple-touch-icon.png'>

Coloca esta etiqueta justo dentro de tu <head>. Para profundizar en las peculiaridades de iOS, revisa nuestra guía de configuración del Apple Touch Icon.

Paso 2: Configura el Web Manifest (Android)

Los dispositivos Android buscan un archivo site.webmanifest. Este archivo JSON le dice a Chrome exactamente cómo debe comportarse tu sitio cuando se guarda en la pantalla de inicio, incluyendo qué ícono usar y si debe ocultar la interfaz del navegador.

Crea un archivo llamado site.webmanifest en tu directorio raíz y agrega este código:

{
  "name": "Nombre de tu App",
  "short_name": "App",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "theme_color": "#ffffff",
  "background_color": "#ffffff",
  "display": "standalone"
}

Nota la propiedad display: standalone. Esta es la línea mágica que elimina la barra de URL cuando un usuario abre tu sitio desde su pantalla de inicio, dándole esa sensación de app nativa.

Paso 3: Vincula el Manifest en tu HTML

Necesitas apuntar tu documento HTML a ese archivo manifest. Agrega esto justo debajo de tu etiqueta de Apple touch icon:

<link rel='manifest' href='/site.webmanifest'>

Errores comunes a evitar

Incluso los desarrolladores senior se equivocan en esto. Aquí están las trampas en las que veo a la gente caer cada semana:

  • Usar SVGs para iOS: Safari todavía no soporta completamente SVG para el ícono de la pantalla de inicio. Quédate con PNG. Lee nuestra guía de favicon SVG para ver dónde pertenecen realmente los SVGs (pista: pestañas de escritorio).
  • Olvidar el color de fondo: Como mencioné, iOS odia la transparencia. Rellena el fondo de tu PNG con un color sólido de tu marca.
  • Pesadillas de caché: Si actualizas tu ícono, iOS se aferrará obstinadamente al antiguo. Agrega una cadena de consulta de versión como ?v=2 a la ruta del archivo en tu HTML para forzar una actualización.

Probando tu implementación

No confíes en las herramientas de desarrollo de tu navegador de escritorio para esto. La única forma real de probar tu ícono de pantalla de inicio es en un dispositivo físico.

Abre tu sitio en un iPhone, toca el botón Compartir y selecciona 'Agregar a la pantalla de inicio'. Mira el ícono de vista previa. Si es tu nítido PNG de 180x180, has ganado. Haz lo mismo en un dispositivo Android usando el menú de Chrome. Si el ícono se ve pixelado, revisa las rutas de tu manifest.
