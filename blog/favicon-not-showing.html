Favicon Not Showing? 7 Common Causes & Fixes

You've added a favicon to your website, but it's not appearing in the browser tab. Frustrating? Let's go through the 7 most common reasons and their fixes, ordered from most to least likely.

1. Browser Cache (Most Common)

Favicons are cached more aggressively than almost anything else in the browser. Even after you update the file, the browser may show the old one (or no icon) for hours or days.

Fix:

  • Hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
  • Open in Incognito/Private mode
  • Clear browser cache: Settings → Privacy → Clear browsing data → Cached images and files
  • Add a cache-busting query string: href="/favicon.ico?v=2"

2. Wrong File Path

The most common coding error. The browser can't find your favicon file because the path is incorrect.

Fix:

  • Use absolute paths starting with /: href="/favicon.ico"
  • Open https://yoursite.com/favicon.ico directly in the browser — can you see the icon?
  • Check for typos in the filename (e.g., favicon.ico vs Favicon.ico — case matters on Linux servers)

3. Missing or Incorrect HTML Tags

The <link> tag might be missing, malformed, or placed outside the <head>.

Fix: Ensure this code is inside your <head> tags:

<link rel="icon" type="image/x-icon" href="/favicon.ico">

Common mistakes:

  • Using rel="shortcut icon" — while still supported, rel="icon" is the standard
  • Missing the type attribute — add type="image/x-icon" or type="image/png"
  • Placing the link tag in <body> instead of <head>

4. Corrupt or Invalid File

The file might look fine on your computer but be corrupted or in the wrong format.

Fix:

  • Verify the file is a valid ICO/PNG — open it in an image viewer
  • Don't just rename a JPEG to .ico — that doesn't convert the format
  • Regenerate using a proper tool like Mzu favicondl's Image to Favicon

5. Server Not Serving the Correct MIME Type

Some servers don't recognize .ico files and serve them with the wrong Content-Type header.

Fix: Check the response headers in DevTools (Network tab). The MIME type should be:

  • .ico → image/x-icon or image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

For Apache, add to .htaccess:

AddType image/x-icon .ico

For Nginx:

location ~* \.ico$ {
    add_header Content-Type image/x-icon;
}

6. HTTPS / Mixed Content Issues

If your site is on HTTPS but the favicon URL uses HTTP, the browser will block it silently.

Fix:

  • Use protocol-relative paths: href="/favicon.ico" (no http:// or https://)
  • Or explicitly use HTTPS: href="https://yoursite.com/favicon.ico"

7. Favicon Not Showing in Google Search Results

Your favicon appears in the browser tab but not in Google search results. This is a separate issue.

Google's favicon requirements:

  • The favicon URL must be crawlable (not blocked by robots.txt)
  • Size must be a multiple of 48px (e.g., 48×48, 96×96, 144×144)
  • Must be a square image
  • Google won't display favicons that it deems inappropriate or low quality
  • It can take days to weeks for Google to pick up a new favicon

Fix: Use the Google Search Console URL Inspection tool to request re-crawling of your homepage.

Quick Debugging Checklist

  1. Open https://yoursite.com/favicon.ico — does it load?
  2. Open DevTools → Network tab → reload → filter by "favicon" — any 404 errors?
  3. Check the HTML source — is the <link> tag correct and inside <head>?
  4. Try Incognito mode — does it show there?
  5. Check another browser — is it browser-specific?

你已经给网站添加了 Favicon，但浏览器标签页里就是不显示。很烦对吧？让我们看看最常见的 7 个原因及其解决方法，按可能性从高到低排列。

1. 浏览器缓存（最常见）

Favicon 的缓存比几乎所有其他资源都更激进。即使你更新了文件，浏览器可能还会显示旧的（或不显示）长达数小时甚至数天。

解决方法：

  • 强制刷新：Ctrl+Shift+R（Windows/Linux）或 Cmd+Shift+R（Mac）
  • 在无痕/隐私模式中打开
  • 清除浏览器缓存：设置 → 隐私 → 清除浏览数据 → 缓存的图片和文件
  • 添加缓存破坏查询字符串：href="/favicon.ico?v=2"

2. 文件路径错误

最常见的代码错误。浏览器找不到你的 Favicon 文件因为路径不正确。

解决方法：

  • 使用以 / 开头的绝对路径：href="/favicon.ico"
  • 直接在浏览器打开 https://你的网站.com/favicon.ico — 能看到图标吗？
  • 检查文件名拼写（如 favicon.ico vs Favicon.ico — 在 Linux 服务器上大小写敏感）

3. HTML 标签缺失或错误

<link> 标签可能缺失、格式错误、或放在了 <head> 外面。

解决方法：确保以下代码在 <head> 标签内：

<link rel="icon" type="image/x-icon" href="/favicon.ico">

常见错误：

  • 使用 rel="shortcut icon" — 虽然仍然支持，但 rel="icon" 才是标准
  • 缺少 type 属性 — 添加 type="image/x-icon" 或 type="image/png"
  • 把 link 标签放在了 <body> 而不是 <head> 中

4. 文件损坏或格式无效

文件在你的电脑上看起来没问题，但可能已损坏或格式错误。

解决方法：

  • 确认文件是有效的 ICO/PNG — 用图片查看器打开它
  • 不要只是把 JPEG 重命名为 .ico — 那并不会转换格式
  • 使用正规工具重新生成，如 Mzu favicondl 的图片转 Favicon

5. 服务器未正确提供 MIME 类型

有些服务器不识别 .ico 文件，会用错误的 Content-Type 头来提供。

解决方法：在 DevTools（网络标签）中检查响应头。MIME 类型应该是：

  • .ico → image/x-icon 或 image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

6. HTTPS / 混合内容问题

如果你的网站使用 HTTPS 但 Favicon URL 使用 HTTP，浏览器会静默阻止。

解决方法：

  • 使用协议无关路径：href="/favicon.ico"（不要加 http:// 或 https://）
  • 或明确使用 HTTPS：href="https://你的网站.com/favicon.ico"

7. Google 搜索结果中不显示 Favicon

你的 Favicon 在浏览器标签页中显示但不在 Google 搜索结果中。这是一个单独的问题。

Google 的 Favicon 要求：

  • Favicon URL 必须可抓取（不被 robots.txt 阻止）
  • 尺寸必须是 48px 的倍数（如 48×48、96×96、144×144）
  • 必须是正方形图片
  • Google 不会显示它认为不合适或低质量的 Favicon
  • Google 获取新 Favicon 可能需要几天到几周

解决方法：使用 Google Search Console 的 URL 检查工具请求重新抓取你的首页。

快速排查清单

  1. 打开 https://你的网站.com/favicon.ico — 能加载吗？
  2. 打开 DevTools → 网络标签 → 刷新 → 筛选 "favicon" — 有 404 错误吗？
  3. 检查 HTML 源码 — <link> 标签正确且在 <head> 内吗？
  4. 试试无痕模式 — 那里能显示吗？
  5. 换个浏览器 — 是浏览器特定问题吗？

ウェブサイトに favicon を追加したのに、ブラウザのタブに表示されない。イライラしますよね？最も可能性の高いものから低いものへと順に、7つのよくある原因とその解決策を見ていきましょう。

1. ブラウザのキャッシュ (最も一般的)

favicon は、ブラウザで他のほとんどのものよりも積極的にキャッシュされます。ファイルを更新した後でも、ブラウザは数時間から数日間、古いもの (またはアイコンなし) を表示することがあります。

解決策:

  • スーパーリロード (ハードリフレッシュ): Ctrl+Shift+R (Windows/Linux) または Cmd+Shift+R (Mac)
  • シークレット/プライベートモードで開く
  • ブラウザのキャッシュをクリア: 設定 → プライバシー → 閲覧履歴データの消去 → キャッシュされた画像とファイル
  • キャッシュ無効化のためのクエリ文字列を追加: href="/favicon.ico?v=2"

2. ファイルパスが間違っている

最もよくあるコーディングエラーです。パスが間違っているため、ブラウザが favicon ファイルを見つけられません。

解決策:

  • / で始まる絶対パスを使用する: href="/favicon.ico"
  • ブラウザで https://yoursite.com/favicon.ico を直接開いてみる — アイコンが見えますか？
  • ファイル名のタイプミスを確認する (例: favicon.ico と Favicon.ico — Linux サーバーでは大文字と小文字が区別されます)

3. HTML タグの欠落または間違い

<link> タグが欠落しているか、形式が間違っているか、<head> の外に配置されている可能性があります。

解決策: このコードが <head> タグ内に含まれていることを確認してください:

<link rel="icon" type="image/x-icon" href="/favicon.ico">

よくある間違い:

  • rel="shortcut icon" の使用 — まだサポートされていますが、rel="icon" が標準です
  • type 属性の欠落 — type="image/x-icon" または type="image/png" を追加してください
  • <head> ではなく <body> に link タグを配置している

4. ファイルの破損または無効なファイル

ファイルはあなたのコンピューター上では問題なく見えるかもしれませんが、破損しているか、間違った形式である可能性があります。

解決策:

  • ファイルが有効な ICO/PNG であることを確認する — 画像ビューアで開いてみてください
  • JPEG を単に .ico にリネームしないでください — それでは形式は変換されません
  • Mzu favicondl の Image to Favicon のような適切なツールを使用して再生成する

5. サーバーが正しい MIME タイプを配信していない

一部のサーバーは .ico ファイルを認識せず、間違った Content-Type ヘッダーで配信することがあります。

解決策: DevTools (Network タブ) でレスポンスヘッダーを確認してください。MIME タイプは次のようになっているはずです:

  • .ico → image/x-icon または image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

Apache の場合、.htaccess に以下を追加します:

AddType image/x-icon .ico

Nginx の場合:

location ~* \.ico$ {
    add_header Content-Type image/x-icon;
}

6. HTTPS / 混在コンテンツの問題

サイトが HTTPS で運用されているのに、favicon の URL が HTTP を使用している場合、ブラウザはそれをサイレントにブロックします。

解決策:

  • プロトコル相対パスを使用する: href="/favicon.ico" (http:// または https:// なし)
  • または明示的に HTTPS を使用する: href="https://yoursite.com/favicon.ico"

7. Google 検索結果に favicon が表示されない

ブラウザのタブには favicon が表示されるのに、Google 検索結果には表示されない。これは別の問題です。

Google の favicon 要件:

  • favicon の URL はクロール可能であること (robots.txt でブロックされていないこと)
  • サイズは48px の倍数であること (例: 48×48, 96×96, 144×144)
  • 正方形の画像であること
  • Google が不適切または低品質と判断した favicon は表示されません
  • 新しい favicon が Google に認識されるまでには数日から数週間かかることがあります

解決策: Google Search Console の URL 検査ツールを使用して、ホームページの再クロールをリクエストしてください。

웹사이트에 favicon을 추가했지만, 브라우저 탭에 나타나지 않습니까? 답답하시죠? 가장 가능성이 높은 것부터 낮은 순서로, 가장 흔한 7가지 원인과 해결 방법을 살펴보겠습니다.

1. 브라우저 캐시 (가장 흔함)

favicon은 브라우저에서 다른 어떤 것보다 훨씬 적극적으로 캐시됩니다. 파일을 업데이트한 후에도 브라우저는 몇 시간 또는 며칠 동안 이전 아이콘(또는 아이콘 없음)을 표시할 수 있습니다.

해결 방법:

  • 하드 새로고침: Ctrl+Shift+R (Windows/Linux) 또는 Cmd+Shift+R (Mac)
  • 시크릿/개인 모드에서 열기
  • 브라우저 캐시 지우기: 설정 → 개인 정보 보호 → 인터넷 사용 기록 삭제 → 캐시된 이미지 및 파일
  • 캐시 무효화 쿼리 문자열 추가: href="/favicon.ico?v=2"

2. 잘못된 파일 경로

가장 흔한 코딩 오류입니다. 경로가 올바르지 않아 브라우저가 favicon 파일을 찾을 수 없습니다.

해결 방법:

  • /로 시작하는 절대 경로를 사용하세요: href="/favicon.ico"
  • 브라우저에서 https://yoursite.com/favicon.ico를 직접 열어보세요 — 아이콘이 보이나요?
  • 파일 이름의 오타를 확인하세요 (예: favicon.ico vs Favicon.ico — Linux 서버에서는 대소문자를 구분합니다)

3. HTML 태그 누락 또는 오류

<link> 태그가 누락되었거나, 형식이 잘못되었거나, <head> 외부에 배치되었을 수 있습니다.

해결 방법: 다음 코드가 <head> 태그 안에 있는지 확인하세요:

<link rel="icon" type="image/x-icon" href="/favicon.ico">

흔한 실수:

  • rel="shortcut icon" 사용 — 여전히 지원되지만, rel="icon"이 표준입니다
  • type 속성 누락 — type="image/x-icon" 또는 type="image/png"를 추가하세요
  • link 태그를 <head> 대신 <body>에 배치

4. 손상되었거나 유효하지 않은 파일

파일이 컴퓨터에서는 정상적으로 보일 수 있지만, 손상되었거나 잘못된 형식일 수 있습니다.

해결 방법:

  • 파일이 유효한 ICO/PNG인지 확인하세요 — 이미지 뷰어에서 열어보세요
  • JPEG 파일을 단순히 .ico로 이름만 바꾸지 마세요 — 그렇게 한다고 형식이 변환되지 않습니다
  • Mzu favicondl의 Image to Favicon과 같은 적절한 도구를 사용하여 다시 생성하세요

5. 서버가 올바른 MIME 타입을 제공하지 않음

일부 서버는 .ico 파일을 인식하지 못하고 잘못된 Content-Type 헤더로 제공할 수 있습니다.

해결 방법: DevTools (네트워크 탭)에서 응답 헤더를 확인하세요. MIME 타입은 다음과 같아야 합니다:

  • .ico → image/x-icon 또는 image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

Apache의 경우, .htaccess에 다음을 추가하세요:

AddType image/x-icon .ico

Nginx의 경우:

location ~* \.ico$ {
    add_header Content-Type image/x-icon;
}

6. HTTPS / 혼합 콘텐츠 문제

사이트가 HTTPS를 사용하는데 favicon URL이 HTTP를 사용하면 브라우저가 이를 자동으로 차단합니다.

해결 방법:

  • 프로토콜 상대 경로를 사용하세요: href="/favicon.ico" (http:// 또는 https:// 없음)
  • 또는 명시적으로 HTTPS를 사용하세요: href="https://yoursite.com/favicon.ico"

7. Google 검색 결과에 favicon이 표시되지 않음

브라우저 탭에는 favicon이 나타나지만 Google 검색 결과에는 나타나지 않습니다. 이는 별개의 문제입니다.

Google의 favicon 요구사항:

  • favicon URL은 크롤링 가능해야 합니다 (robots.txt에 의해 차단되지 않아야 함)
  • 크기는 48px의 배수여야 합니다 (예: 48×48, 96×96, 144×144)
  • 정사각형 이미지여야 합니다
  • Google은 부적절하거나 품질이 낮다고 판단하는 favicon을 표시하지 않습니다
  • 새로운 favicon이 Google에 반영되기까지 며칠에서 몇 주가 걸릴 수 있습니다

해결 방법: Google Search Console의 URL 검사 도구를 사용하여 홈페이지 재크롤링을 요청하세요.

Has añadido un favicon a tu sitio web, pero no aparece en la pestaña del navegador. ¿Frustrante? Repasemos las 7 razones más comunes y sus soluciones, ordenadas de la más a la menos probable.

1. Caché del navegador (más común)

Los favicons se almacenan en caché de forma más agresiva que casi cualquier otra cosa en el navegador. Incluso después de actualizar el archivo, el navegador puede mostrar el antiguo (o ningún icono) durante horas o días.

Solución:

  • Actualización forzada: Ctrl+Shift+R (Windows/Linux) o Cmd+Shift+R (Mac)
  • Abrir en modo Incógnito/Privado
  • Borrar caché del navegador: Configuración → Privacidad → Borrar datos de navegación → Imágenes y archivos en caché
  • Añadir una cadena de consulta para evitar la caché: href="/favicon.ico?v=2"

2. Ruta de archivo incorrecta

El error de codificación más común. El navegador no puede encontrar tu archivo favicon porque la ruta es incorrecta.

Solución:

  • Usa rutas absolutas que empiecen con /: href="/favicon.ico"
  • Abre https://yoursite.com/favicon.ico directamente en el navegador — ¿puedes ver el icono?
  • Verifica si hay errores tipográficos en el nombre del archivo (por ejemplo, favicon.ico vs Favicon.ico — en servidores Linux, las mayúsculas y minúsculas importan)

3. Etiquetas HTML faltantes o incorrectas

La etiqueta <link> podría faltar, estar mal formada o colocada fuera del <head>.

Solución: Asegúrate de que este código esté dentro de tus etiquetas <head>:

<link rel="icon" type="image/x-icon" href="/favicon.ico">

Errores comunes:

  • Usar rel="shortcut icon" — aunque todavía es compatible, rel="icon" es el estándar
  • Falta el atributo type — añade type="image/x-icon" o type="image/png"
  • Colocar la etiqueta link en <body> en lugar de <head>

4. Archivo corrupto o inválido

El archivo podría parecer correcto en tu computadora, pero estar corrupto o en el formato incorrecto.

Solución:

  • Verifica que el archivo sea un ICO/PNG válido — ábrelo en un visor de imágenes
  • No te limites a renombrar un JPEG a .ico — eso no convierte el formato
  • Regenera usando una herramienta adecuada como Image to Favicon de Mzu favicondl

5. El servidor no sirve el tipo MIME correcto

Algunos servidores no reconocen los archivos .ico y los sirven con la cabecera Content-Type incorrecta.

Solución: Revisa las cabeceras de respuesta en DevTools (pestaña Red). El tipo MIME debería ser:

  • .ico → image/x-icon o image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

Para Apache, añade a .htaccess:

AddType image/x-icon .ico

Para Nginx:

location ~* \.ico$ {
    add_header Content-Type image/x-icon;
}

6. Problemas de HTTPS / Contenido mixto

Si tu sitio está en HTTPS pero la URL del favicon usa HTTP, el navegador lo bloqueará silenciosamente.

Solución:

  • Usa rutas relativas al protocolo: href="/favicon.ico" (sin http:// ni https://)
  • O usa HTTPS explícitamente: href="https://yoursite.com/favicon.ico"

7. Favicon no aparece en los resultados de búsqueda de Google

Tu favicon aparece en la pestaña del navegador pero no en los resultados de búsqueda de Google. Este es un problema aparte.

Requisitos de favicon de Google:

  • La URL del favicon debe ser rastreable (no bloqueada por robots.txt)
  • El tamaño debe ser un múltiplo de 48px (por ejemplo, 48×48, 96×96, 144×144)
  • Debe ser una imagen cuadrada
  • Google no mostrará favicons que considere inapropiados o de baja calidad
  • Google puede tardar de días a semanas en recoger un nuevo favicon

Solución: Usa la herramienta de inspección de URL de Google Search Console para solicitar un nuevo rastreo de tu página de inicio.

Lista de verificación rápida para depuración

  1. Abre https://yoursite.com/favicon.ico — ¿carga?
  2. Abre DevTools → pestaña Red → recarga → filtra por "favicon" — ¿hay errores 404?
  3. Revisa el código fuente HTML — ¿la etiqueta <link> es correcta y está dentro de <head>?
  4. Prueba el modo Incógnito — ¿se muestra allí?
  5. Verifica en otro navegador — ¿es un problema específico del navegador?
