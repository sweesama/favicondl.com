Favicon Not Showing? 7 Common Causes & Fixes

You've added a favicon to your website, but it's not appearing in the browser tab. Frustrating? Let's go through the 7 most common reasons and their fixes, ordered from most to least likely.

1. Browser Cache (Most Common)

Favicons are cached more aggressively than almost anything else in the browser. Even after you update the file, the browser may show the old one (or no icon) for hours or days.

Fix:

  • Hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
  • Open in Incognito/Private mode
  • Clear browser cache: Settings → Privacy → Clear browsing data → Cached images and files
  • Add a cache-busting query string: href="/favicon.ico?v=2"

2. Wrong File Path

The most common coding error. The browser can't find your favicon file because the path is incorrect.

Fix:

  • Use absolute paths starting with /: href="/favicon.ico"
  • Open https://yoursite.com/favicon.ico directly in the browser — can you see the icon?
  • Check for typos in the filename (e.g., favicon.ico vs Favicon.ico — case matters on Linux servers)

3. Missing or Incorrect HTML Tags

The <link> tag might be missing, malformed, or placed outside the <head>.

Fix: Ensure this code is inside your <head> tags:

<link rel="icon" type="image/x-icon" href="/favicon.ico">

Common mistakes:

  • Using rel="shortcut icon" — while still supported, rel="icon" is the standard
  • Missing the type attribute — add type="image/x-icon" or type="image/png"
  • Placing the link tag in <body> instead of <head>

4. Corrupt or Invalid File

The file might look fine on your computer but be corrupted or in the wrong format.

Fix:

  • Verify the file is a valid ICO/PNG — open it in an image viewer
  • Don't just rename a JPEG to .ico — that doesn't convert the format
  • Regenerate using a proper tool like Mzu favicondl's Image to Favicon

5. Server Not Serving the Correct MIME Type

Some servers don't recognize .ico files and serve them with the wrong Content-Type header.

Fix: Check the response headers in DevTools (Network tab). The MIME type should be:

  • .ico → image/x-icon or image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

For Apache, add to .htaccess:

AddType image/x-icon .ico

For Nginx:

location ~* \.ico$ {
    add_header Content-Type image/x-icon;
}

6. HTTPS / Mixed Content Issues

If your site is on HTTPS but the favicon URL uses HTTP, the browser will block it silently.

Fix:

  • Use protocol-relative paths: href="/favicon.ico" (no http:// or https://)
  • Or explicitly use HTTPS: href="https://yoursite.com/favicon.ico"

7. Favicon Not Showing in Google Search Results

Your favicon appears in the browser tab but not in Google search results. This is a separate issue.

Google's favicon requirements:

  • The favicon URL must be crawlable (not blocked by robots.txt)
  • Size must be a multiple of 48px (e.g., 48×48, 96×96, 144×144)
  • Must be a square image
  • Google won't display favicons that it deems inappropriate or low quality
  • It can take days to weeks for Google to pick up a new favicon

Fix: Use the Google Search Console URL Inspection tool to request re-crawling of your homepage.

Quick Debugging Checklist

  1. Open https://yoursite.com/favicon.ico — does it load?
  2. Open DevTools → Network tab → reload → filter by "favicon" — any 404 errors?
  3. Check the HTML source — is the <link> tag correct and inside <head>?
  4. Try Incognito mode — does it show there?
  5. Check another browser — is it browser-specific?

你已经给网站添加了 Favicon，但浏览器标签页里就是不显示。很烦对吧？让我们看看最常见的 7 个原因及其解决方法，按可能性从高到低排列。

1. 浏览器缓存（最常见）

Favicon 的缓存比几乎所有其他资源都更激进。即使你更新了文件，浏览器可能还会显示旧的（或不显示）长达数小时甚至数天。

解决方法：

  • 强制刷新：Ctrl+Shift+R（Windows/Linux）或 Cmd+Shift+R（Mac）
  • 在无痕/隐私模式中打开
  • 清除浏览器缓存：设置 → 隐私 → 清除浏览数据 → 缓存的图片和文件
  • 添加缓存破坏查询字符串：href="/favicon.ico?v=2"

2. 文件路径错误

最常见的代码错误。浏览器找不到你的 Favicon 文件因为路径不正确。

解决方法：

  • 使用以 / 开头的绝对路径：href="/favicon.ico"
  • 直接在浏览器打开 https://你的网站.com/favicon.ico — 能看到图标吗？
  • 检查文件名拼写（如 favicon.ico vs Favicon.ico — 在 Linux 服务器上大小写敏感）

3. HTML 标签缺失或错误

<link> 标签可能缺失、格式错误、或放在了 <head> 外面。

解决方法：确保以下代码在 <head> 标签内：

<link rel="icon" type="image/x-icon" href="/favicon.ico">

常见错误：

  • 使用 rel="shortcut icon" — 虽然仍然支持，但 rel="icon" 才是标准
  • 缺少 type 属性 — 添加 type="image/x-icon" 或 type="image/png"
  • 把 link 标签放在了 <body> 而不是 <head> 中

4. 文件损坏或格式无效

文件在你的电脑上看起来没问题，但可能已损坏或格式错误。

解决方法：

  • 确认文件是有效的 ICO/PNG — 用图片查看器打开它
  • 不要只是把 JPEG 重命名为 .ico — 那并不会转换格式
  • 使用正规工具重新生成，如 Mzu favicondl 的图片转 Favicon

5. 服务器未正确提供 MIME 类型

有些服务器不识别 .ico 文件，会用错误的 Content-Type 头来提供。

解决方法：在 DevTools（网络标签）中检查响应头。MIME 类型应该是：

  • .ico → image/x-icon 或 image/vnd.microsoft.icon
  • .png → image/png
  • .svg → image/svg+xml

6. HTTPS / 混合内容问题

如果你的网站使用 HTTPS 但 Favicon URL 使用 HTTP，浏览器会静默阻止。

解决方法：

  • 使用协议无关路径：href="/favicon.ico"（不要加 http:// 或 https://）
  • 或明确使用 HTTPS：href="https://你的网站.com/favicon.ico"

7. Google 搜索结果中不显示 Favicon

你的 Favicon 在浏览器标签页中显示但不在 Google 搜索结果中。这是一个单独的问题。

Google 的 Favicon 要求：

  • Favicon URL 必须可抓取（不被 robots.txt 阻止）
  • 尺寸必须是 48px 的倍数（如 48×48、96×96、144×144）
  • 必须是正方形图片
  • Google 不会显示它认为不合适或低质量的 Favicon
  • Google 获取新 Favicon 可能需要几天到几周

解决方法：使用 Google Search Console 的 URL 检查工具请求重新抓取你的首页。

快速排查清单

  1. 打开 https://你的网站.com/favicon.ico — 能加载吗？
  2. 打开 DevTools → 网络标签 → 刷新 → 筛选 "favicon" — 有 404 错误吗？
  3. 检查 HTML 源码 — <link> 标签正确且在 <head> 内吗？
  4. 试试无痕模式 — 那里能显示吗？
  5. 换个浏览器 — 是浏览器特定问题吗？
